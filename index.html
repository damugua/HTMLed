HTMLed JavaScript FrameWork

Light weight javascript framework to convert html pages to locale/Language specific pages on the fly for Jquery Mobile. Written by Niranjan(Nirus)

HTML + Rendered = HTMLed.js for Jquery mobile Framework

This light weight javascript framework that converts the Jquery Mobile HTML pages to language specific pages and renders it to the user on the fly. This works with Tokens Identifiers or Place Holder where the strings are replaced to the specified language.

See the below code example or checkout the index.html page on github

<h3>label.description</h3> //"label.description" is a place holder defined in ".properties" file
<p>label.descriptionText</p>"label.descriptionText" is a place holder defined in ".properties" file

This works with the Jquery Mobile framework.

Implementation/Usage:

  • Create a project (sample project is found on Github for reference)
  • Add meta tag content in the head tag <meta charset="UTF-8">

  • Add the "HTMLed.js" before the Jquery Mobile framework tag

      <!--HTMLed.js-->
   <script type="text/javascript" charset="utf-8" src="js/HTMLed.js"></script>

  <!--Jquery Mobile Framework-->
  <script type="text/javascript" src="js/jquerymobile/jquery.mobile-1.3.1.min.js"></script>
  <link rel="stylesheet" href="js/jquerymobile/jquery.mobile-1.3.1.min.css" />

  • Now declare global variable before "HTMLed.js" script tag, set the language preference(You can read the comments in the script file to get better understanding)

    <script>
var global = {
          options : {
                     autoDetect: false,  
                     langPref: "fr",     
                     langBundle:"bundle",                  
            }
        };
    </script>
    <script type="text/javascript" charset="utf-8" src="js/HTMLed.js"></script>

  • Your project is good to go!!

For External HTML pages

  • call "window.HTMLed" with the options as parameter to the method as shown below

    window.HTMLed({
       externalPage : "popup.html",
       options : {
                  autoDetect: false,  
                  langPref: "fr",    
                  langBundle:"bundle", 
                 },
       callBack:function(parsed){
                                 $('#popup-container').html(parsed);
                                 $('#popup-wrapper').popup("open", { transition: "pop" }); 
                                }                                            
            });

    the above option illustrates the example of loading external HTML page as popup

  • Callback in the parameter provides you the flexibility to use the parsed HTML the way you want to implement.

Important pointers on this framework:

".properties" file:

Its basically a JSON file parsed inside the framework. Hence while writing a language specific ".properties" file , write it exactly as a JSON file otherwise your language bundle will not load properly. Strictly follow the JSON formatting.

"autoDetect" - true :

This has to be set inside "HTMLed.js" file It is used to detect the browser specific language settings. If you set it as true, it get's the browser language settings and automatically load it for you. Here is the catch when set as "true" - You should have the ".properties" file name exactly matching the name of the browser language returned. You can use this JSfiddle Link to get or see the language name set for the browser. For example if it returns "en-US" your ".properties" file for that specific language should be named as "en-US.properties". The framework will load it automatically and display the page in that language as specified. This happens irrespective of "langPref" set in "HTMLed.js" script file.

"autoDetect" - false :

If you want to explicitly load the language irrespective of your browser settings, then make "autoDetect" to "false" in the HTMLed.js file. This will load the language file given in "langPref". Just mention as "langPref: en-US", this load "en-US.properties" file.
